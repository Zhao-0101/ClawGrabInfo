# DocManager MVP 产品文档

## 产品概述

DocManager是一款智能文档管理桌面程序，通过分析四级目录结构中的第三级文件夹命名，自动重组文件架构，帮助用户按业务类型而非项目名称管理文档，实现从"按项目存储"到"按类型管理"的范式转换。

## 功能详细说明

### 文件移动功能

#### 功能描述
提供安全、可靠的文件移动能力，是所有重排操作的基础。

#### 技术要点

| 要点 | 说明 |
|-----|------|
| 原子性操作 | 使用临时目录中转，确保操作要么完全成功，要么完全失败 |
| 权限保持 | 保留原文件的读写权限、创建时间、修改时间 |
| 跨卷支持 | 检测源和目标是否在同一磁盘，必要时使用复制+删除 |
| 错误恢复 | 操作失败时自动回滚已完成的移动 |
| 进度反馈 | 大文件移动时显示进度条 |

#### 操作流程
```
1. 用户选择源目录
2. 程序分析目录结构
3. 生成操作计划（移动清单）
4. 用户确认
5. 创建备份快照（用于撤销）
6. 执行移动操作
7. 验证完整性
8. 更新操作日志
```

---

### 智能重排功能（核心算法逻辑）

#### 核心概念

**四级目录结构定义**：
```
Level 1: 根目录/工作区（如：一级/）
Level 2: 项目/主体目录（如：A项目/）
Level 3: 类型/分类目录（如：文档/、图片/）
Level 4: 内容目录（如：2024/、素材/）
```

**重排规则**：提取Level 3的目录名作为新分类依据，将Level 2的项目作为其子项。

#### 算法逻辑

```
输入：源目录路径 source_root
输出：重排计划 reorganize_plan

算法步骤：
1. 遍历 source_root 下的所有 Level 2 目录
2. 对每个 Level 2 目录，获取其下的 Level 3 子目录列表
3. 创建映射表：Level3名称 → [Level2路径列表]
4. 对每个映射项，生成新的目录结构：
   - 新 Level 1: 用户指定的目标根目录
   - 新 Level 2: Level3名称（如：文档、图片）
   - 新 Level 3: 原 Level 2 名称（如：A项目、B项目）
   - 新 Level 4: 原 Level 3 下的所有内容
5. 返回重排计划，包含：
   - 源路径 → 目标路径 的映射列表
   - 冲突检测信息
   - 预估操作时间和空间
```

#### 冲突检测与处理

| 冲突类型 | 检测逻辑 | 处理策略 |
|---------|---------|---------|
| 同名文件夹 | 目标位置已存在同名文件夹 | 选项A：合并；选项B：重命名（自动加后缀）；选项C：跳过 |
| 文件覆盖 | 目标位置已存在同名文件 | 选项A：覆盖；选项B：保留两者（重命名）；选项C：跳过 |
| 循环引用 | 目标目录是源目录的子目录 | 拒绝操作并提示 |
| 权限不足 | 无法读取源或写入目标 | 预检测并提示用户 |
| 磁盘空间 | 跨卷操作时目标空间不足 | 预计算并警告 |

#### 示例执行流程

```
源结构：
/workspace/
  ├── A项目/
  │   ├── 文档/
  │   │   └── 2024/
  │   └── 图片/
  │       └── 素材/
  └── B项目/
      ├── 文档/
      │   └── 2023/
      └── 合同/
          └── 保密协议.pdf

分析阶段：
- Level 3 发现：文档（2次）、图片（1次）、合同（1次）

生成计划：
1. /workspace/A项目/文档 → /workspace/按类型/文档/A项目
2. /workspace/B项目/文档 → /workspace/按类型/文档/B项目
3. /workspace/A项目/图片 → /workspace/按类型/图片/A项目
4. /workspace/B项目/合同 → /workspace/按类型/合同/B项目

目标结构：
/workspace/
  ├── A项目/          (保留原结构，可选删除)
  ├── B项目/
  └── 按类型/
      ├── 文档/
      │   ├── A项目/2024/
      │   └── B项目/2023/
      ├── 图片/
      │   └── A项目/素材/
      └── 合同/
          └── B项目/保密协议.pdf
```

## 非功能需求

### 性能需求

| 指标 | 要求 | 说明 |
|-----|------|------|
| 目录扫描速度 | 10,000个文件夹/秒 | 基于SSD的基准测试 |
| 重排计划生成 | < 3秒 | 10,000个文件夹规模 |
| 文件移动速度 | 接近系统原生速度 | 无额外处理开销 |
| 内存占用 | < 500MB | 处理100,000个文件时 |
| 启动时间 | < 2秒 | 从点击到界面可用 |

### 安全需求

| 需求 | 实现方式 |
|-----|---------|
| 操作可逆 | 每次操作生成快照，支持一键撤销 |
| 数据完整性 | 移动前后校验文件MD5 |
| 权限保护 | 保留原文件ACL权限（Windows）/Unix权限（Linux/Mac） |
| 防误操作 | 危险操作需要二次确认，提供预览模式 |
| 日志审计 | 记录所有操作，便于追溯 |

### 兼容性需求

| 平台 | 支持版本 | 特殊说明 |
|-----|---------|---------|
| Windows | 10/11 | 支持长路径（>260字符） |
| macOS | 12+ | 支持APFS特性 |
| Linux | Ubuntu 20.04+ | 支持ext4/xfs/btrfs |

### 可靠性需求

- **崩溃恢复**：程序崩溃后可从断点恢复（通过操作日志）
- **异常处理**：网络驱动断开、磁盘满等异常 graceful degradation
- **数据备份**：自动创建操作前快照（保留30天）
